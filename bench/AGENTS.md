# BENCHMARK — 3-TIER TOKEN EFFICIENCY

Measures CLI compact output vs raw MCP JSON. Tier 1 (per-response), Tier 2 (E2E agent), Tier 3 (schema overhead).

## STRUCTURE

```
scripts/
  run_real_benchmark.py       # Tier 1 — live MCP, multi-scene, percentile stats
  run_e2e_benchmark.py        # Tier 2 — E2E agent task simulation
  analyze_schema_overhead.py  # Tier 3 — schema token comparison
  run_benchmark.py            # Mock — offline subprocess testing
  mcp_real_scenario.py        # Scenario runner + compact formatter
  e2e_policy_engine.py        # Deterministic policy engine (fact-based)
  e2e_task_defs.py            # Task defs (setup → steps → verify → cleanup)
  bench_scene_helper.py       # Scene switching
  setup/*.sh                  # check_env, start_mcp, smoke_mcp, prepare_project
data/
  scenarios.{real,mock,real.trust}.json  # Scenario definitions
  mocks/mock_unity.py                   # Fake MCP responses (offline)
reports/
  tier{1,2,3}-*.json                    # Raw results
  final-benchmark-report.md             # Consolidated human-readable
```

## WHERE TO LOOK

| Task | File |
|------|------|
| Run Tier 1 | `scripts/run_real_benchmark.py --scene-profile all --iterations 10` |
| Run Tier 3 | `scripts/analyze_schema_overhead.py --url http://127.0.0.1:8080/mcp` |
| New E2E task | `scripts/e2e_task_defs.py` — follow existing dict structure |
| New scenario | `data/scenarios.real.json` — tool_name, params, expected fields |
| Read results | `reports/final-benchmark-report.md` |

## CONVENTIONS

- Scripts standalone: `python3 bench/scripts/<script>.py`
- Stable RNG: `random.Random(f"{mode}:{scenario}:{iteration}")`
- Token counting: `len(json.dumps(data))` as proxy
- Stats: median, p5, p95, bootstrap CI

## KEY NUMBERS

| Tier 1 output reduction | **92.8%** median (79.3–98.4%) |
|---|---|
| Tier 2 macro avg | 2.4% (schema overhead dominates) |
| Tier 3 schema | MCP 14,877 vs CLI 480 tokens (**31x**) |

## NOTES

- Tier 1/3 require live Unity + MCP at `http://127.0.0.1:8080/mcp`
- Bench scenes auto-generated by `unity-project/Assets/Editor/BenchSceneBuilder.cs`
- E2E policy engine is deterministic (no LLM) — simulates agent via fact extraction
